# สรุปการทำงานของเมนู กำหนดเงื่อนไขการอนุมัติผลการสอบอัตโนมัติ (/ExamResultCondition)

## 1. จุดประสงค์ของระบบ (Overview)
จากหน้าจอและ Source code เมนู "กำหนดเงื่อนไขการอนุมัติผลการสอบอัตโนมัติ" มีไว้สำหรับ **ตั้งกฎและเงื่อนไข (Rules & Conditions)** เพื่อให้ระบบดำเนินการ **อนุมัติผลการสอบให้กับรอบสอบต่างๆ โดยอัตโนมัติ** แทนที่เจ้าหน้าที่จะต้องมานั่งตรวจสอบและกดอนุมัติด้วยตนเองทีละรอบสอบ ระบบจะทำงานตามช่วงเวลาที่กำหนดและเกณฑ์ของแต่ละศูนย์สอบ ทำให้กระบวนการออกผลสอบเป็นไปอย่างรวดเร็วและเป็นระบบมากขึ้น

## 2. การทำงานและฟังก์ชันหลัก (Workflow & Functions)

ฟังก์ชันการทำงานจะแบ่งออกเป็นการกำหนดข้อมูลทั่วไป และการกำหนดเงื่อนไขแยกตามศูนย์สอบ 2 รูปแบบ ดังนี้:

### 2.1 ข้อมูลทั่วไปของเงื่อนไข
*   **วันที่เริ่มต้น (Start Date) - วันที่สิ้นสุด (End Date):** สำหรับกำหนดช่วงวันที่ที่กฎ/เงื่อนไขนี้จะมีผลบังคับใช้งาน
*   **รายละเอียด (Description):** บันทึกคำอธิบายว่าทำไมถึงตั้งเงื่อนไขนี้ (Remark)
*   **สถานะการใช้งาน (Status):** สามารถเลือกเปิดใช้งาน (Enable) หรือปิดการใช้งาน (Disable) เงื่อนไขแต่ละตัวได้

### 2.2 การจัดการเงื่อนไขระดับ "ศูนย์สอบ" (แบ่งเป็น 2 ประเภท)

**ประเภทที่ 1: ศูนย์สอบที่ระบบอนุมัติผลการสอบอัตโนมัติทั้งหมด (Approve All)**
*   เป็นการระบุรายชื่อ "ศูนย์สอบ" เข้าไปใน List แบบให้ความไว้วางใจสูงสุด
*   หากมีการจัดการสอบโดยศูนย์สอบที่อยู่ในรายชื่อนี้ ระบบจะ**อนุมัติผลสอบให้ทันที 100%** แบบอัตโนมัติ ไม่ว่าจะเหลือกี่คนหรือผ่านกี่คนก็ตาม

**ประเภทที่ 2: ศูนย์สอบที่ระบบอนุมัติผลการสอบอัตโนมัติตาม Option ที่ถูกระบุไว้ (Approve by Condition)**
*   เป็นการระบุรายชื่อ "ศูนย์สอบ" แบบมีเงื่อนไขผูกมัด ระบบจะอนุมัติผลสอบอัตโนมัติ **ก็ต่อเมื่อ** ภาพรวมผลการสอบของรอบสอบนั้นๆ ตรงตามเกณฑ์ที่ระบุไว้
*   **เงื่อนไข (Option) ที่ตั้งค่าได้:** 
    *   ระบุเป็น "รอบสอบที่มีจำนวนผู้สอบผ่านเกิน `[ x ] %` แต่ไม่เกิน `[ y ] %` ของจำนวนผู้สอบทั้งหมด"
    *   *(ตัวอย่างเช่น: ตั้งค่าไว้ที่ ผู้สอบผ่านเกิน 40% แต่ไม่เกิน 80% ถ้ารอบสอบไหนที่ผลการสอบออกมามีคนผ่านอยู่ในช่วงเปอร์เซ็นต์นี้ ระบบก็จะดำเนินการอนุมัติผลสอบให้ทันที หากไม่ตรงตามเกณฑ์ ก็จะถูกพักไว้ให้เจ้าหน้าที่ตรวจสอบและอนุมัติเอง)*

### 2.3 ฟังก์ชันการจัดการอื่นๆ 
*   **การเพิ่มศูนย์สอบ:** มีระบบเลือกศูนย์สอบ (Dropdown) แบบค้นหาได้ และมีตัวเลือก `Select All` สำหรับดึงศูนย์สอบทั้งหมดเข้ามาในเงื่อนไขได้เลย
*   **หน้าจอค้นหา (Search):** สามารถค้นหากฎการอนุมัติที่มีอยู่ในระบบ ตามช่วงวันที่และสถานะ
*   **Audit Tracking:** ในหน้าจอ Detail จะมีการแสดงข้อมูล **ผู้ที่แก้ไขล่าสุด** และ **วัน/เวลาที่แก้ไขล่าสุด** ไว้ด้านล่าง เพื่อให้สามารถตรวจสอบย้อนหลังได้ว่าใครเป็นคนตั้งค่ากฎนั้นๆ 

## สรุป
หน้าจอนี้ทำหน้าที่เป็นหน้าจอ Administrator Configuration สำหรับควบคุมพฤติกรรมของระบบ Auto-Approve Result โดยทำหน้าที่บันทึก "กฎ" (Rule) บันทึกลงใน Database จากนั้นระบบ Background Process หรือตอนที่ประมวลผลคะแนนสอบเสร็จสิ้น จะมาอ่านค่าจากตารางนี้ (ผ่าน BL: `IIQE_ExamResultConditionBL`) เพื่อพิจารณาว่ารอบสอบนั้นจะได้ไฟเขียวผ่านการอนุมัติเลยหรือไม่
